## Постановка задачи

На основе требований из `bilateral.pdf` реализован 9-точечный bilateral-фильтр для полутонового BMP-изображения:

- версия на CPU;
- версия на GPU (CUDA);
- замер времени выполнения на CPU и GPU;
- сохранение результатов фильтрации в BMP.

Границы обрабатываются по правилу **nearest pixel** (clamp), как требуется в задании.

## Что сделано

В ноутбуке `HPC_Bilateral_6131.ipynb`:

- загружается входное изображение `input1.bmp` в градациях серого;
- выполняется bilateral-фильтрация 3x3 на CPU;
- выполняется bilateral-фильтрация 3x3 на GPU (Numba CUDA);
- выводится время CPU и GPU, а также ускорение;
- сохраняются:
  - `output_cpu.bmp`
  - `output_gpu.bmp` (если доступна CUDA).

## Входные данные

Согласно условию:

- входное полутоновое изображение в BMP (`input1.bmp`);
- параметры фильтра:
  - `SIGMA_D` — параметр пространственной Gaussian-составляющей;
  - `SIGMA_R` — параметр range-составляющей.

Параметры задаются в отдельной ячейке ноутбука.

## Выходные данные

Ноутбук формирует:

- время обработки на CPU;
- время обработки на GPU (если CUDA доступна);
- изображения результата в BMP;
- проверку различий между CPU и GPU (максимальное и среднее абсолютное отличие).

## Как запустить

1. Положить `input1.bmp` в папку `Bilateral`.
2. Открыть `HPC_Bilateral_6131.ipynb`.
3. Установить зависимости (при необходимости):

```bash
pip install numpy pillow numba
```

4. Запустить все ячейки по порядку.

## Примечание по требованию texture memory

В исходном PDF отдельно указано использование texture memory в CUDA-реализации.  
В данном ноутбуке реализован практический вариант через Numba CUDA для запуска из Python/Jupyter.  
Если нужна строго C++/CUDA-реализация именно с texture memory (`texture<>`, `tex2D`), можно добавить отдельную версию `.cu`.
